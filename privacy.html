Privacy Policy

Quick Search File is built around local search. The app indexes folders and connected locations you choose so you can find files by filename, extension, folder path, and search terms.

Last updated: May 16, 2026

English日本語繁體中文

Scope of Data Collection

Quick Search File creates local indexes for file search. Depending on the features you use, the app may process filenames, file extensions, folder paths, file identifiers, file sizes, modified dates, selected folder information, search terms, index update status, app settings, subscription status, and related metadata needed to display search results.

The app does not collect your name, phone number, contacts, location information, or other personally identifiable information for advertising purposes.

Purpose of File Access

Quick Search File uses the iOS, iPadOS, or macOS file picker and platform permissions to let you actively authorize specific folders. Folder access is used only to read file names, file extensions, paths, and related metadata needed to build and update a search index.

Google Drive Read-Only Access

If you choose to connect Google Drive, Quick Search File uses Google OAuth and the https://www.googleapis.com/auth/drive.readonly permission scope. This read-only scope allows the app to view Google Drive files and folders that your Google account can access.

Quick Search File uses Google Drive access to list and read file and folder metadata needed for search, indexing, result display, and user-directed file access. The app does not use Google Drive access to create, edit, move, delete, or share your Google Drive files.

You can revoke Google Drive access at any time from your Google Account security settings.

OneDrive Access

If you choose to connect Microsoft OneDrive, Quick Search File uses Microsoft OAuth to access OneDrive file and folder information needed for search, indexing, result display, and user-directed file access.

Quick Search File uses OneDrive access only for file search features. The app does not use OneDrive access to create, edit, move, delete, or share your OneDrive files.

You can revoke OneDrive access through your Microsoft account permissions or remove the connected account in the app when that option is available.

Dropbox Access

If you choose to connect Dropbox, Quick Search File uses Dropbox OAuth and Dropbox API permissions such as account_info.read, files.metadata.read, and files.content.read.

Dropbox access is used to show the connected account, list folders, read file and folder metadata needed for search and indexing, display search results, and download a file only when you choose to open it from a search result. The app does not use Dropbox access to create, edit, move, delete, or share your Dropbox files.

You can revoke Dropbox access through your Dropbox connected-app settings or remove the connected account in the app when that option is available.

Google API Limited Use Disclosure

Quick Search File's use and transfer of information received from Google APIs will adhere to the Google API Services User Data Policy, including the Limited Use requirements.

Google user data is used only to provide or improve user-facing file search features. It is not sold, used for advertising, transferred for unrelated purposes, or used to train generalized artificial intelligence or machine learning models.

Data Storage and Use

Local index data and search settings are stored on your device. Quick Search File does not upload your local file index, local search history, or local folder metadata to a Quick Search File server.

If the macOS Command Line Interface (CLI) is enabled, it uses the same app-authorized local index. CLI output may include filenames, paths, extensions, file sizes, and other metadata you request in text, paths-only, or JSON form.

Subscriptions and Payments

If you use in-app subscriptions, payment and subscription management are handled by the Apple App Store. Quick Search File may receive subscription status needed to unlock app features, but it does not obtain or store your credit card information.

Data Sharing

Quick Search File does not sell your personal information. Data may be shared only when needed to provide user-requested functionality, comply with law, protect security, process App Store subscriptions through Apple, operate hosting infrastructure, or when you explicitly export, copy, open, or share information yourself.

Data Deletion

You can delete the app to remove locally stored index data and settings. Google Drive access can be revoked from your Google Account permissions, OneDrive access can be revoked from your Microsoft account permissions, Dropbox access can be revoked from your Dropbox connected-app settings, and related indexes can be removed or refreshed in the app where available.

Website Data

This website is a static product website hosted on GitHub Pages. Standard server logs or platform-level analytics may process basic request information such as IP address, browser, referring page, and request time for security, reliability, and hosting operations.

Policy Updates

This Privacy Policy may be updated due to app changes, Google Drive support, OneDrive support, Dropbox support, storage provider changes, subscriptions, platform requirements, or legal requirements. Updated versions will be posted on this page.

Contact

If you have questions about this Privacy Policy, please use the Quick Search File support page at https://quicksearchfile.com/support.html.

プライバシーポリシー

収集するデータの範囲

Quick Search File は、ファイル検索のために端末内で索引を作成します。利用する機能に応じて、ファイル名、拡張子、フォルダパス、ファイル識別子、ファイルサイズ、更新日時、選択したフォルダ情報、検索語、索引更新状態、App設定、サブスクリプション状態、および検索結果の表示に必要な関連メタデータを処理する場合があります。

本Appは、広告目的で氏名、電話番号、連絡先、位置情報、その他個人を識別できる情報を収集しません。

ファイルアクセスの目的

Quick Search File は、iOS / iPadOS / macOS のファイルピッカーおよびプラットフォーム権限を使用し、ユーザーが特定のフォルダを明示的に許可できるようにします。フォルダアクセスは、検索索引の作成と更新に必要なファイル名、拡張子、パス、および関連メタデータを読み取る目的のみに使用されます。

Google Drive の読み取り専用アクセス

Google Drive を接続する場合、Quick Search File は Google OAuth と https://www.googleapis.com/auth/drive.readonly 権限スコープを使用します。この読み取り専用スコープにより、App は Google アカウントでアクセス可能な Google Drive のファイルとフォルダを表示できます。

Quick Search File は、検索、索引作成、検索結果表示、およびユーザーが指示したファイルアクセスのために、Google Drive のファイルおよびフォルダのメタデータを一覧表示・読み取りします。本Appは、Google Drive のファイルを作成、編集、移動、削除、共有するために Google Drive アクセスを使用しません。

Google Drive へのアクセスは、Google アカウントのセキュリティ設定からいつでも取り消すことができます。

OneDrive へのアクセス

Microsoft OneDrive を接続する場合、Quick Search File は Microsoft OAuth を使用し、検索、索引作成、検索結果表示、およびユーザーが指示したファイルアクセスに必要な OneDrive のファイルとフォルダ情報へアクセスします。

Quick Search File は、OneDrive へのアクセスをファイル検索機能のためにのみ使用します。本Appは、OneDrive のファイルを作成、編集、移動、削除、共有するために OneDrive アクセスを使用しません。

OneDrive へのアクセスは、Microsoft アカウントの権限設定、またはApp内で利用可能な場合は接続済みアカウントの削除によって取り消すことができます。

Dropbox へのアクセス

Dropbox を接続する場合、Quick Search File は Dropbox OAuth と account_info.read、files.metadata.read、files.content.read などの Dropbox API 権限を使用します。

Dropbox アクセスは、接続アカウントの表示、フォルダ一覧、検索と索引作成に必要なファイルおよびフォルダのメタデータの読み取り、検索結果の表示、およびユーザーが検索結果からファイルを開いた場合のダウンロードにのみ使用されます。本Appは、Dropbox のファイルを作成、編集、移動、削除、共有するために Dropbox アクセスを使用しません。

Dropbox へのアクセスは、Dropbox の接続済みApp設定、またはApp内で利用可能な場合は接続済みアカウントの削除によって取り消すことができます。

Google API Limited Use の開示

Quick Search File が Google API から受け取る情報の使用および転送は、Limited Use 要件を含む Google API Services User Data Policy に従います。

Google ユーザーデータは、ユーザー向けのファイル検索機能を提供または改善する目的のみに使用されます。販売、広告利用、無関係な目的への転送、または汎用的なAI・機械学習モデルの訓練には使用されません。

データの保存と利用

ローカル索引データと検索設定は端末内に保存されます。Quick Search File は、ローカルのファイル索引、ローカル検索履歴、ローカルフォルダのメタデータを Quick Search File のサーバーへアップロードしません。

macOS の Command Line Interface (CLI) を有効にした場合、CLI は同じApp承認済みローカル索引を使用します。CLI出力には、ユーザーが要求したファイル名、パス、拡張子、ファイルサイズ、およびその他のメタデータがテキスト、パスのみ、または JSON 形式で含まれる場合があります。
