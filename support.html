Frequently Asked Questions

Why do I need to authorize folders?

Quick Search File needs permission before it can index file names and extensions. On iOS and iPadOS, folder access is granted through the system Files picker. On macOS, access is handled by macOS permissions and app settings.

How does Google Drive support work?

When Google Drive support is enabled, Quick Search File uses Google OAuth with the drive.readonly scope. This allows the app to view Google Drive file and folder metadata for search and indexing. The app does not create, edit, move, delete, or share your Google Drive files.

How does OneDrive support work?

When OneDrive support is enabled, Quick Search File uses Microsoft OAuth to access OneDrive file and folder information needed for search and indexing. The app does not create, edit, move, delete, or share your OneDrive files.

Does the app upload my local search index?

No. Local index data and search settings are stored on your device and are not uploaded to a Quick Search File server.

What should I do if search results are outdated?

Refresh or update the index in the app after adding, deleting, moving, or renaming many files. Cloud provider sync state can also affect whether files are available for indexing.

Why can some cloud folders not be indexed?

Some storage providers, app folders, shared drives, read-only locations, or cloud-only files may restrict third-party access. Provider permissions and local availability can affect results.

How do I revoke Google Drive or OneDrive access?

You can revoke Google Drive access from your Google Account security settings and OneDrive access from your Microsoft account permissions. After revoking access, remove or refresh related indexes in the app where available.

よくある質問

なぜフォルダの許可が必要ですか？

Quick Search File がファイル名と拡張子を索引化するには権限が必要です。iOS / iPadOS では、フォルダアクセスはシステムの Files ピッカーを通じて許可されます。macOS では、macOS の権限と App 設定によってアクセスが管理されます。

Google Drive 対応はどのように動作しますか？

Google Drive 対応を有効にすると、Quick Search File は Google OAuth と drive.readonly スコープを使用します。これにより、App は検索と索引作成のために Google Drive のファイルとフォルダのメタデータを表示できます。本Appは、Google Drive のファイルを作成、編集、移動、削除、共有しません。

OneDrive 対応はどのように動作しますか？

OneDrive 対応を有効にすると、Quick Search File は Microsoft OAuth を使用し、検索と索引作成に必要な OneDrive のファイルとフォルダ情報へアクセスします。本Appは、OneDrive のファイルを作成、編集、移動、削除、共有しません。

App はローカル検索索引をアップロードしますか？

いいえ。ローカル索引データと検索設定は端末内に保存され、Quick Search File のサーバーへアップロードされません。

検索結果が古い場合はどうすればよいですか？

ファイルを大量に追加、削除、移動、名前変更した後は、App 内で索引を更新してください。クラウドプロバイダの同期状態も、ファイルが索引化可能かどうかに影響する場合があります。

一部のクラウドフォルダを索引化できないのはなぜですか？

一部のストレージプロバイダ、App フォルダ、共有ドライブ、読み取り専用の場所、またはクラウド上のみのファイルは、第三者Appからのアクセスが制限される場合があります。プロバイダの権限とローカルでの利用可能性が結果に影響します。

Google Drive または OneDrive へのアクセスを取り消すには？

Google Drive へのアクセスは Google アカウントのセキュリティ設定から、OneDrive へのアクセスは Microsoft アカウントの権限設定から取り消せます。取り消し後、App 内で関連する索引を削除または更新してください。

常見問題

為什麼需要授權資料夾？

Quick Search File 需要權限才能建立檔案名稱與副檔名索引。在 iOS / iPadOS 上，資料夾存取會透過系統 Files 檔案選擇器授權。在 macOS 上，存取權限由 macOS 權限與 App 設定管理。

Google Drive 支援如何運作？

啟用 Google Drive 支援時，Quick Search File 會使用 Google OAuth 與 drive.readonly 權限範圍。這允許 App 為搜尋與索引目的查看 Google Drive 檔案與資料夾 metadata。本 App 不會建立、編輯、移動、刪除或分享您的 Google Drive 檔案。

OneDrive 支援如何運作？

啟用 OneDrive 支援時，Quick Search File 會使用 Microsoft OAuth，以存取搜尋與索引所需的 OneDrive 檔案與資料夾資訊。本 App 不會建立、編輯、移動、刪除或分享您的 OneDrive 檔案。

App 會上傳我的本機搜尋索引嗎？

不會。本機索引資料與搜尋設定會儲存在您的裝置上，不會上傳到 Quick Search File 伺服器。

搜尋結果不是最新時該怎麼辦？

大量新增、刪除、移動或重新命名檔案後，請在 App 內重新整理或更新索引。雲端服務的同步狀態也可能影響檔案是否可被索引。

為什麼有些雲端資料夾不能索引？

部分儲存空間提供者、App 資料夾、共享雲端硬碟、唯讀位置或僅存在雲端的檔案，可能會限制第三方 App 存取。提供者權限與本機可用狀態都可能影響結果。

如何撤銷 Google Drive 或 OneDrive 存取？

您可以從 Google 帳號安全性設定撤銷 Google Drive 存取權限，並從 Microsoft 帳號權限設定撤銷 OneDrive 存取權限。撤銷後，請在 App 內可用時移除或重新整理相關索引。
