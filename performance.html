Measured search performance for large local file sets.

The numbers below were measured against generated local test data using combined filename terms with default AND matching and an extension filter. Quick Search File is built to keep local file search in the millisecond range on both desktop and mobile devices.

Test conditions

100,000 generated files. Combined terms with default AND matching.

The stress test used multiple filename terms matched together by default, plus an extension filter. Each platform result is the average of three repeated searches.

Interpretation

Use the numbers as search-time evidence, not a universal guarantee.

The purpose of this page is to show search timing at a known dataset scale without overpromising performance on every device or storage provider.

What this measures

These results measure the app search path after a local search index is ready for use.

Device and storage can vary

Actual search time can vary by hardware, storage state, cloud file availability, index size, and system load.
