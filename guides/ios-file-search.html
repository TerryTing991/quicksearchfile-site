How to search files on iPhone and iPad with Quick Search File

The fastest iOS search workflow starts with one rule: choose or connect the locations you want indexed, then search file names and extensions from a local index.

1. Choose folders through the Files picker

iOS protects file access by design. Quick Search File uses the system Files picker, so you explicitly choose the folders the app can index. This can include folders you create in On My iPhone, iCloud Drive, and Files app locations that allow third-party folder access.

2. Connect Google Drive or OneDrive when needed

When Google Drive support is enabled, Quick Search File uses Google OAuth with the drive.readonly scope so it can view Drive file and folder metadata for search. When OneDrive support is enabled, Quick Search File uses Microsoft OAuth to access OneDrive file and folder information needed for search and indexing. The app does not use Google Drive or OneDrive access to create, edit, move, delete, or share your cloud files.

3. Build or refresh the local index

The app creates a local filename index for the folders and connected locations you authorize. Refresh the index after major file changes so results stay current.

4. Search by filename terms and extension

Use one filename term, multiple terms, or an extension filter such as pdf, docx, or xlsx. Extension search is useful when you remember the file type but not the exact name.

FAQ

Can Quick Search File scan every folder on my iPhone automatically?

No. iOS requires users to authorize folders through the Files picker, connect Google Drive with read-only permission, or connect OneDrive through Microsoft OAuth. Quick Search File indexes only locations you choose or connect.

Can every cloud provider folder be selected?

No. Some app folders, read-only locations, and cloud providers may not allow folder access. Google Drive may use read-only Google Drive API access, and OneDrive may use Microsoft OAuth when you connect your account.

Does Quick Search File upload my file list?

No. The index is stored locally on your device and is used for search and result display.
